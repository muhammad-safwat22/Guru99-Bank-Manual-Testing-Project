# 🏦 Guru99 Bank Manual Testing Project

> **End-to-End Manual Testing Portfolio Project**
>
> A complete Manual Testing project covering the full **Software Testing Life Cycle (STLC)**,
> including **Requirement Analysis, Mockup Review, Test Planning, Test Case Design,
> Test Execution, Defect Reporting, Confirmation Testing, Regression Testing,
> Change Request Analysis, Test Closure, and Jira Project Management using Agile Kanban.**

------------------------------------------------------------------------

# 📖 Project Overview

This repository contains a complete **Manual Testing** project
for the **Guru99 Bank Application**.

The project follows industry-standard QA practices, beginning with reviewing
the **Software Requirements Specification (SRS)**
and **System Mockups**, followed by preparing a **System Test Plan**,
designing and executing manual test cases, reporting and tracking defects,
performing **Confirmation Testing** and **Regression Testing**,
analyzing requirement changes, preparing a **Test Closure Report**,
and managing testing activities using **Atlassian Jira Cloud**
with an **Agile Kanban** workflow.

------------------------------------------------------------------------

# 🎯 Project Objectives

-   Review and analyze software requirements.
-   Review system mockups before designing test cases.
-   Prepare a professional System Test Plan.
-   Design comprehensive manual test cases.
-   Execute functional, validation, and negative test scenarios.
-   Report and track software defects.
-   Perform Confirmation Testing.
-   Perform Regression Testing.
-   Analyze requirement changes across SRS versions.
-   Prepare Test Closure documentation.
-   Manage testing activities using Jira Cloud.
